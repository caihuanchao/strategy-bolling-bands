# 自适应多周期布林带交易工作台 - 美的集团(000333)

## Problem Statement
我们如何构建一个专业的交易系统，通过**多周期共振确认** + **自适应布林带参数** + **完整的工作台工具链**，帮助有经验的交易者对 美的集团(000333) 进行纪律化的实盘交易？

## Recommended Direction

构建一个**渐进式迭代**的交易工作台：

**阶段 1（MVP）：** 
- 日线级别布林带策略（固定参数起步）
- 基础回测功能
- 简单信号提醒（控制台/日志文件）

**阶段 2：**
- 引入 4小时/1小时 多周期共振
- 增加 MACD/RSI 辅助确认

**阶段 3：**
- 自适应参数（根据波动率动态调整）
- Web 仪表盘（Streamlit）
- 完整交易日志与绩效分析

核心设计理念：**先跑通，再优化。** 每一步都要可独立运行并产生价值。

## Key Assumptions to Validate

- [ ] **数据源可获取性** - AKShare 或其他数据源能提供美的集团(000333)的日线/小时线数据（测试失败是网络问题，不是数据源不存在）
- [ ] **布林带在 美的集团(000333) 上有效** - 先做简单回测，验证 (20,2) 参数在历史上的表现
- [ ] **多周期共振提升效果** - A/B 测试：单周期 vs 多周期的胜率/盈亏比对比
- [ ] **自适应参数优于固定参数** - 滚动窗口计算波动率，动态调整 N 和 M

## MVP Scope

**包含：**
1. 数据获取模块（支持 AKShare，预留扩展接口）
2. 基础布林带计算（N=20, M=2）
3. 信号生成（买入/卖出/观望）
4. 回测引擎（计算历史收益、最大回撤、胜率）
5. 简单的信号输出（控制台打印 + CSV 日志）

**暂不包含：**
- 实时监控（先做历史回测验证策略）
- 自适应参数（阶段 3）
- 多周期共振（阶段 2）
- Web UI（阶段 3）

## Not Doing (and Why)

- **不做自动化下单对接券商** - 券商 API 接入复杂度高，先解决信号问题
- **不做多标的支持** - 专注把美的集团(000333)做精，后续再扩展
- **不过早优化参数** - 先用标准 (20,2)，有数据后再优化
- **不做复杂的机器学习模型** - 保持策略透明可解释，布林带已经够强了

## Open Questions

1. **回测时间范围？** 用最近 1 年的数据。
2. **交易成本？** 需要考虑交易成本（佣金、印花税）。
3. **仓位管理？** 每次交易固定金额，还是固定股数。
4. **数据源备选方案？** 如果 AKShare 不稳定，备用数据源为BaoStock。
