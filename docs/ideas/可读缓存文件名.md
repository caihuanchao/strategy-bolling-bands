# 可读缓存文件名

## Problem Statement

**How Might We:** 让缓存文件名更具可读性，通过在文件名中包含股票名称，方便人类一眼识别文件对应的股票？

## Recommended Direction

采用最直接的方案，在文件名中前置股票名称：

- **主数据缓存**：`{name}_{symbol}_{period}_{start_date}.csv`
  - 示例：`美的集团_000333_daily_20250101.csv`
- **布林带缓存**：`{name}_{symbol}.csv`
  - 示例：`美的集团_000333.csv`

### 具体改动点：

1. **修改 `src/data_fetcher.py`**：
   - `get_cache_path()` 和 `get_cache_metadata_path()` 增加 `name` 参数
   - `get_stock_data()` 增加可选 `name` 参数，优先用传入的 name，没有则用 `config.symbol_name`
   - `load_from_cache()` 需要支持查找新旧两种文件名（先找新格式，找不到找旧格式）
   - 新增 `migrate_cache_files()` 函数，启动时自动迁移旧文件

2. **修改 `src/cache.py`**：
   - `save_bollinger_history()` 使用新文件名格式
   - `load_bollinger_history()` 支持查找新旧两种文件名
   - 新增 `migrate_bollinger_cache()` 函数

3. **名称清理规则**：
   - 特殊字符（空格、*、ST、括号等）替换为下划线
   - 示例：`*ST平安` → `_ST平安`

## Key Assumptions to Validate

- [x] **中文文件名支持**：当前环境是 macOS，没问题；Windows 可能需要测试，但用户已确认接受
- [x] **name 参数可用性**：
  - 从 config 有 `symbol_name`
  - 从 `fetch_batch_data()` 有 `StockInfo.name`
- [ ] **自动迁移时间可接受**：用户确认接受自动迁移

## MVP Scope

**In Scope:**
- 修改缓存路径生成函数，支持新格式
- 自动迁移旧缓存文件到新格式
- 名称特殊字符清理（下划线替换）

**Out of Scope (for now):**
- 拼音 fallback（用户直接选了中文）
- 不用兼容（读写操作只用新格式）

## Not Doing (and Why)

- **不做名称后置** — 用户明确选了名称前置
- **不做复杂分隔符** — 下划线已经够用，保持简单
- **不修改 watchlist 以外的缓存** — 只处理有 name 的情况
