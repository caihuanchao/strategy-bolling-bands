# 自适应参数实验室

## Problem Statement

如何让用户在 Web 仪表盘中手动选择市场状态并切换对应的布林带预设参数，同时提供滚动回测工具验证参数效果，从而比固定参数 (N=20, M=2.0) 获得更匹配当前市场环境的信号？

## Recommended Direction

**参数实验室（两阶段迭代）**：在现有 Web 仪表盘新增「参数实验室」标签页。

**B1 — 参数预设切换（MVP，先做）：**

- 4 组预设参数模板，对应不同市场状态：趋势跟踪 (20, 2.0)、震荡市 (10, 1.5)、高波动 (30, 2.5)、收紧 (15, 1.8)
- 用户选择模板后，可手动微调 N（5-50）和 M（1.0-3.0）
- 页面实时展示参数变化前后的信号数量对比（买入/卖出信号数变化），但不自动应用——用户手动点「应用」后全量信号更新
- 增加「当前市场特征速览」面板：展示全市场平均带宽、波动率变化趋势，辅助用户判断，不做自动判断

**B2 — 单标的滚动回测（增强，后续）：**

- 用户在参数实验室中选择一只股票，系统对该标的历史数据做滚动窗口回测
- 搜索空间粗粒度：N 步长 5，M 步长 0.25
- 输出：N×M 胜率热力图 + 最优参数组合 + 与固定参数 (20, 2.0) 的绩效对比
- 明确标注「历史回测不代表未来表现」

## Key Assumptions to Validate

- [ ] **用户能正确判断市场状态** — 用「市场特征速览」面板辅助，观察用户是否会参考面板数据做选择
- [ ] **滚动回测性能可接受** — 先在单标的上跑，确保 3 秒内出结果，达不到则进一步缩窄搜索空间
- [ ] **参数切换产生可见差异** — 切换预设后信号数量/质量应有明显变化，否则功能无价值

## MVP Scope (B1)

**包含：**

1. 新的「参数实验室」Tab 页（与现有「今日信号」「全部概览」并列）
2. 4 组预设参数模板的下拉选择器
3. N 和 M 的手动滑块（N: 5-50 步长 1，M: 1.0-3.0 步长 0.1）
4. 参数变化前后的信号数量对比卡片（买入数/卖出数变化）
5. 「应用参数」按钮，点击后用新参数重新计算全量信号
6. 市场特征速览面板：全市场平均带宽、平均波动率趋势（基于现有 std 和 bandwidth 数据）
7. 后端 API：`/api/params` GET 获取当前参数，`/api/params` POST 更新参数并重算信号

**技术约束：**

- 参数切换后需重新计算所有股票的布林带和信号，用后台线程执行避免阻塞
- 参数变更不持久化到 config.py，刷新页面后恢复默认 (20, 2.0)——保持简单

## Not Doing (and Why)

- **不做自动市场状态识别（ADX/ATR）** — 用户要手动控制，自动识别与需求矛盾
- **不做多标的并行回测** — B2 阶段先单标的，确保交互流畅
- **不做 Walk-Forward 验证** — 太复杂，留给后续
- **不做参数持久化** — MVP 阶段不保存自定义参数，保持简单
- **不做 MACD/RSI 参数优化** — 本期仅布林带 N 和 M
- **不做仓位/止损参数优化** — 交易参数属于另一个问题域

## Open Questions

1. 预设参数的 N/M 值是直接用上表的固定值，还是先对 美的集团(000333) 做一次简单回测来确定？建议先固定，B2 阶段用数据验证。
2. 参数切换后是否需要同时更新多周期（4h/1h）的参数？建议 MVP 只改日线，多周期后续再对齐。
3. 市场特征速览面板的数据来源——是用全市场 35 只股票的平均值，还是只看当前选中标的？建议全市场平均，给出整体市场环境感知。
