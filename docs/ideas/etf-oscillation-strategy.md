# ETF 震荡波段策略

## 问题陈述

**How Might We** 实现一个专为 ETF 震荡市设计的短线波段策略：先判断市场是否真震荡，再基于布林带 + RSI + K 线形态产生买卖信号，并在震荡环境破坏时及时预警退出？

## 推荐方向

新建 `EtfOscillationStrategy`（策略 ID: `etf_oscillation`），参照 `KdjBollingerAtrStrategy` 的三阶段架构，完整实现文档中的"布林带 + RSI 均值回归 + K 线形态确认 + 失效预警"逻辑。

与现有 `BollingerStrategy` 的关键区别：本策略**先判断环境再交易**——均线走平、ATR 收缩、箱体确认三者满足才认为震荡有效。信号需要 BB 触轨 + RSI 极端 + K 线形态三重确认，而非简单触碰即触发。

## 实现范围

### MVP（本次实现）

| 模块 | 内容 |
|------|------|
| `src/indicators.py` | 新增 4 个 K 线形态识别函数：锤子线、射击之星、看涨吞没、看跌吞没 |
| `src/strategies/etf_oscillation.py` | 策略主类，三阶段：环境检测 → 信号生成 → 失效预警 |
| `src/strategies/etf_oscillation_interpreter.py` | 中文解读器，6 维解读（环境/RSI/布林带/K线/预警/建议） |
| `app.py` | 注册策略（1 行 import + 1 行 register） |

### 阶段一：环境检测

- MA20 走平判断（5 日斜率 < 0.5%）
- 箱体确认（回溯 60 日内至少 3 次触及支撑/阻力 ±2%）
- ATR 趋势（5 日变化率，稳定或下降视为震荡特征）
- 输出枚举：`oscillation`（震荡）/ `transition`（过渡）/ `trend`（趋势）

### 阶段二：信号生成

仅在 `env = oscillation` 时触发信号。信号分两级：

**B 级（基础信号）：** BB 触轨 + RSI 极端值，两个条件同时满足即产生信号
- 买入：收盘价 <= 布林下轨 + RSI(14) <= 35
- 卖出：收盘价 >= 布林上轨 + RSI(14) >= 65

**A 级（增强信号）：** B 级基础上叠加 K 线形态确认，升级为 A 级
- 买入增强：+ 锤子线或看涨吞没
- 卖出增强：+ 射击之星或看跌吞没

ETF 布林带默认参数 (20, 2.5)，区别于个股的 (20, 2.0)。

### 阶段三：失效预警

以下任一触发即产生卖出信号（平仓离场）：
1. 布林带宽 20 日最高 → 突破在即
2. ADX(14) 从 < 20 突破 25 → 趋势确认
3. MA20/MA60 从缠绕转为发散（5 日内斜率差 > 1%）

### 策略参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| `bb_n` | 20 | 10-30 | 布林带周期 |
| `bb_m` | 2.5 | 1.5-3.0 | 布林带倍数（ETF 宽轨） |
| `rsi_period` | 14 | 9-21 | RSI 周期 |
| `rsi_oversold` | 35 | 25-40 | RSI 超卖阈值 |
| `rsi_overbought` | 65 | 60-75 | RSI 超买阈值 |
| `adx_threshold` | 25 | 20-30 | 失效预警 ADX 阈值 |

### 预设模板

| ID | 标签 | 参数 | 说明 |
|----|------|------|------|
| `etf_standard` | ETF 标准 | bb_m=2.5, rsi=35/65 | 适合宽基 ETF，中等波动 |
| `etf_wide` | 宽轨保守 | bb_m=3.0, rsi=30/70 | 高波动品种，减少假信号 |
| `etf_narrow` | 窄轨积极 | bb_m=2.0, rsi=40/60 | 低波动红利 ETF，更快捕捉 |

## 不做的（及原因）

- **网格交易** — 独立策略，文档第三部分留待后续单独实现
- **分时 MACD 辅助** — 需要多周期数据（15/30 分钟线），当前数据框架不支持，且属锦上添花
- **仓位管理（箱体上下沿仓位比例）** — 属于回测/执行层面，策略本身只负责信号生成
- **与现有 BollingerStrategy 合并** — 两套逻辑定位不同（无条件触碰 vs 震荡环境确认），强行合并增加复杂度

## 关键假设待验证

- [ ] K 线形态识别算法在 A 股 ETF 上的准确率 — 用历史数据人工抽查 50 个形态信号
- [ ] ETF 布林带 (20, 2.5) 的触轨频率是否合理 — 回测对比 (20, 2.0) vs (20, 2.5) 的触轨次数
- [ ] 环境检测的"箱体至少 3 次触及"条件是否过严 — 统计 513910 历史数据中满足条件的时间占比
- [ ] ADX > 25 作为失效预警信号是否及时 — 对比 ADX 突破日 vs 实际趋势启动日的滞后天数

## 已决策

- **信号触发范围**：仅在 `env = oscillation`（震荡）环境下触发信号。`transition` 和 `trend` 环境不生成交易信号。
- **K 线形态角色**：加分项（增强信号等级）。BB 触轨 + RSI 极端 → B 级基础信号；叠加形态确认 → A 级增强信号。
