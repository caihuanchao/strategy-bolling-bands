# 布林带策略工作台 - 200只自选股版

## Context
用户已经有了一个针对美的集团的布林带策略 MVP。现在想要：
1. **扩展到 200 只自选股**，每天收盘后批量扫描
2. **增加成交量放大验证层**，过滤假信号
3. **生成静态 HTML 仪表板**，一目了然看今日信号
4. **用 CSV 维护自选股**，格式：代码 + 名称

## 问题陈述
How Might We 构建一个高效的工具，让用户每天收盘后能快速从 200 只股票里筛选出带成交量验证的布林带信号？

## 推荐方向

### 核心设计理念
- **「先跑通，再优化」**：分两步走
  - Step 1：200只批量扫描 + 静态HTML仪表板
  - Step 2：加入成交量验证层，标注「普通信号」vs「增强信号」

- **极简实用主义**：你自己用，不需要花里胡哨的 UI
  - CSV维护自选股
  - 一键运行脚本
  - 输出静态HTML，浏览器打开看

### 关键决策
| 决策项 | 选择 | 理由 |
|--------|------|------|
| 仪表板技术 | 静态HTML + Plotly图表 | 简单、不需要启动服务器，双击打开就能看 |
| 验证层 | 成交量放大（相对于前N日均量） | 用户明确指定，简单有效过滤假突破 |
| 数据获取 | 沿用现有多源 fallback + 缓存 | 已经验证可行，不需要重写 |

## MVP 范围（Step 1）

### 做什么
1. **自选股管理模块**：读 watchlist.csv（列：symbol, name）
2. **批量数据获取**：遍历200只，依次获取，自动重试，加间隔防限流
3. **信号扫描**：每只股票算布林带，找今日信号
4. **HTML 仪表板**：
   - 页面1：今日信号汇总表（买入/卖出分开）
   - 页面2：所有股票概览（布林带位置、距离、K线小图）
   - 页面3：单只股票详情（点进去看）

### 不做什么（重点！）
- ❌ 实时监控（每天收盘后算一遍就行）
- ❌ 自动下单（不碰券商API）
- ❌ Streamlit / Web服务器（静态HTML足够）
- ❌ 组合管理/仓位建议（先解决「找信号」）
- ❌ 多周期共振（以后再加）
- ❌ 复杂UI/登录/共享（你自己用）

## 增强功能（Step 2）

### 成交量验证层
对于每一个突破信号，额外检查：
- 成交量相比前20日均量，放大了X%（可配置，默认50%或100%）
- 在仪表板上标注：🔴 普通信号 vs 🟢 增强信号（通过验证的）

### 可选增强
- 信号历史记录（昨天有什么信号，今天怎么了）
- 单只股票简单回测（看看布林带在这只票上历史表现）

## 关键假设
- [ ] 「200只批量获取，AKShare 不会被限流」—— 我们加请求间隔（比如每只0.5-1秒）
- [ ] 「成交量验证确实能提升信号质量」—— MVP先加上，看效果
- [ ] 「静态HTML + Plotly 足够好用」—— 先验证流程跑通，不行再升级

## 风险和缓解
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| AKShare 限流导致200只要很久 | 高 | 加随机间隔，支持断点续传，优先用缓存 |
| 数据获取失败太多 | 中 | 每只独立失败不影响整体，最后报告哪些失败了 |
| 静态HTML文件太大打不开 | 低 | 分文件，或只展示有信号的票的图 |
